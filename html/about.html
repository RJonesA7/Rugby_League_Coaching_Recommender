Rugby League Recommender System

About

What this tool does

This web app produces coaching recommendations for an upcoming match based on an input profile of the opposition. The input is expressed as z-scores (standardised values), so a slider value of 0 represents “league average”, +1 means “one standard deviation above average”, and −1 means “one standard deviation below average”.

The output is a ranked set of statistics or stat-themes that the system estimates are most worth targeting against that specific opposition profile.

How it works

1) Retrieve similar opposition cases

Given your opposition input, the system finds historical matches with a similar opposition profile using a nearest neighbour search through a database of over 8000 NRL matches.

2) Fit a weighted model on the fly

Using the retrieved historical cases, the app fits a weighted regression model. More similar cases receive more weight, so the fitted model is tailored to the specific opposition context.

3) Turn explanations into recommendations

The model’s predictions are explained using SHAP contribution values. The explanations for the most positive predictions made by the model are averaged, and returned as recommendations.

Player level

In player level mode, the same logic is applied per position group (middles, halves, etc.). The system retrieves similar opposition position-group cases and fits matchup conditioned models, producing recommendations per position group.

How to interpret the numbers

  • Z-scores (inputs): relative to season averages; + means the opposition tends to do more of that thing than average.
  • Recommendation scores (outputs): relative importance weights from the explanation layer; larger magnitude means more influence.
  • Sign: positive vs negative indicates direction of influence under the fitted model, not a guaranteed causal effect.

Limitations

  • Not causal: recommendations are based on patterns in historical data, not controlled experiments.
  • Feature dependence: many rugby statistics are correlated, which can make attribution methods less stable.
  • Context loss: match events, injuries, weather, and tactics aren’t fully captured in box-score stats.
  • “Similarity” is a choice: the nearest-neighbour feature set defines what “similar opposition” means.
